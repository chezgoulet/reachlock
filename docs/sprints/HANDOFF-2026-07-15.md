## The owner's direction (from this session, in their words distilled)

1. **Ships are crewed machines.** Systems are operated by a body at a
   station (player, NPC/LLM, or an automation module in a hull slot) — see
   `docs/SHIPS.md` §1. Each station gets its own *view* of the same live
   world: pilot = Star Fox view; tactical = targeting HUD whose shots
   render in the flight view; same for mining and scanning.
2. **The FTL-cryo loop is signature gameplay** (SHIPS.md §3): humans must
   reach pods before the jump window; androids/robots (player- or
   LLM-driven) run the crossing. "Our embrace of LLMs and their fail
   states as first-class citizens is going to create emergent gameplay."
3. **Damage lands inside the ship**: broken systems, fires that spread and
   pull crew off stations, power triage in engineering (SHIPS.md §4).
4. **Every hull class expresses the same grammar** at different scale —
   fighter to capital (SHIPS.md §2, §7).
5. **The art bar is set**: ALttP/Chrono Trigger/Pokémon G/S sprite feel,
   Stardew-scale environments rich with detail, lore-derived character
   design. Match it in everything new.

## Watch list / known gaps (be honest about these in your PRs)

- **Needs a playtest pass:** wing sweep angle + canopy glow on the flight
  model (`spawn_loup_garou_model`), interior prop density, camera zoom feel,
  cryo-pod spacing, ladder placement UX.
- **No way to leave the helm**: OnBoard → SpaceFlight exists (pilot seat);
  the reverse (stand up from the helm back to the cockpit) doesn't. The
  docking comment about a `B` walk-ship key was never implemented.
- Crew on the deck you're not on freeze (their shift target isn't in the
  active layout — `route()` returns empty). Fine for now; real cross-deck
  routing should come with the crisis/coordination work.
- `ORDER_ROOMS` (crew order panel) doesn't include the new ship rooms
  (Cockpit/TechBay/Scanner/MedBay/Cryo) yet.
- Landed stations still use the seeded generator (correct per design —
  only ships are authored).
- The lower-deck bridge label vs the station "ADMIN" reuse of
  `RoomKind::Bridge` is fine but easy to trip over when adding room kinds:
  `room_label`, `room_kind_color`, `floor_texture`, and `spawn_props` all
  match exhaustively — the compiler will walk you to every site.
- `generate_hull_interior` (seeded ship layouts) is now unused by the
  client but kept in core — intended for NPC/derelict ships later.
